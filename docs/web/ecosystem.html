Recommended lab layout

  • Network edge — BPI-R3 Mini running CyberThreatGotchi systemd service
  • Assessment Pi — Raspberry Pi with Bjorn + bjorn_bridge.py for CTG webhooks
  • Bench laptop — CTG simulation + CrackBot wordlist work
  • Pocket — Cardputer with PlatformIO or MicroPython status client

Integration paths

CTG pushes via CTG_WEBHOOK_URL or gets polled at /api/status. No hard dependency between repos — wire them when your lab is ready.
